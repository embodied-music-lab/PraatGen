# Recommended: ggml-base.en.bin (English, good speed/accuracy balance)
#
# IMPORTANT NOTES:
#
# 1. CREATION REQUIRES ARGUMENTS
#    Despite the catalogue listing it without parameter expansion,
#    Create SpeechRecognizer takes two string arguments: model filename
#    and language. Calling without arguments produces:
#      "Command not available for current selection"
#    Provenance: Paste Commands, 25 March 2026.
#
# 2. SELECTION STATE FOR CREATION
#    Create SpeechRecognizer may fail if other objects are selected.
#    Clear selection before calling:
#      selectObject: soundId
#      minusObject: soundId
#      recognizerId = Create SpeechRecognizer: "ggml-base.en.bin", "English"
#    Provenance: Empirical debugging, 25 March 2026.
#
# 3. nocheck MUST NOT BE USED ON CREATION
#    nocheck on Create SpeechRecognizer causes silent failure — no object
#    created, no error raised, interpreter variable state may be corrupted.
#    Use fileReadable() to check model path before calling:
#      modelPath$ = preferencesDirectory$ + "/models/whispercpp/" + model$
#      if not fileReadable (modelPath$)
#          # fall back to manual pathway
#      endif
#    Provenance: Empirical debugging, 25 March 2026. See also master prompt
#    memory entry on nocheck interpreter corruption.
#
# 4. Recognize sound WRITES TO INFO WINDOW
#    Every call to Recognize sound clears and overwrites the Info window.
#    Capture the transcript immediately after each call:
#      selectObject: recognizerId, segSoundId
#      Recognize sound
#      transcript$ = info$ ()
#    When processing multiple segments, store transcripts in an array
#    and rebuild any Info window log after the recognition loop.
#    Provenance: Empirical observation, 25 March 2026.
#
# 5. SUSTAINED VOWELS PRODUCE [BLANK_AUDIO]
#    Whisper returns "[BLANK_AUDIO]" for non-speech segments such as
#    sustained vowel phonation (/a/). Filter these before any text
#    matching or sentence merging stage.
#    Provenance: CAPE-V auto-segmenter testing, 25 March 2026.
#
# 6. CREATE ONCE, REUSE
#    The model loads on creation. Pair the recognizer with different
#    Sound objects for each segment rather than creating/destroying
#    repeatedly. This is a significant performance optimization.

  # Cross-type: SpeechRecognizer & Sound (1 command)

  Recognize sound
  # Verified: Recognize sound (no colon, no arguments)
  # Provenance: Paste Commands, 25 March 2026.
  # CRITICAL: The catalogue lists this as "Transcribe" — that is WRONG.
  #   The actual command name is "Recognize sound".
  # Requires: both SpeechRecognizer and Sound selected
  # Output: writes transcript text to Info window (capture via info$())
  # Side effect: clears Info window before writing
  # Does NOT create any new objects
  # Does NOT support nocheck (untested, avoid per Note 3)
