# ============================================================================
# SECTION 1: EDITOR SCRIPTING SYNTAX
# ============================================================================
#
# To send commands to an open editor window from a script:
#
#   editor: objectId
#       # editor commands here
#   endeditor
#
# Or by name:
#
#   editor: "Sound mySound"
#       # editor commands here
#   endeditor
#
# The editor must already be open (via View & Edit).
# Commands inside the block correspond to menu items in the editor.
#
# SCOPING:
# - Inside editor:/endeditor, only editor commands are available
# - Object window commands (selectObject:, Get mean:, etc.) are NOT available
# - To return to the Objects window, use endeditor
# - You can alternate between editor: and endeditor blocks
# - Variable assignments (e.g., pitch = Get pitch) work inside editor blocks
#   and the variable persists after endeditor
#
# EDITOR TARGETING (hard — empirically verified):
#
# The editor: command targets the EDITOR, not the object. Which object
# ID to use depends on which editor type is open:
#
#   | Editor type     | Created by selecting     | Target with       |
#   |-----------------|--------------------------|-------------------|
#   | SoundEditor     | Sound → View & Edit      | editor: soundId   |
#   | TextGridEditor  | Sound + TextGrid → V & E | editor: gridId    |
#
# In a TextGridEditor, the TextGrid is the PRIMARY object. The Sound
# is secondary. Using editor: soundId will HANG — "Editor N does not
# exist" — because the editor is registered under the TextGrid, not
# the Sound.
#
#   # CORRECT (TextGridEditor):
#   selectObject: soundId, gridId
#   View & Edit
#   editor: gridId                 # target the TextGrid
#       Mute channels: "1 2 3"     # Sound command — works fine
#   endeditor
#
#   # WRONG (TextGridEditor):
#   editor: soundId                # HANGS — no editor under Sound
#
# Both Sound-area commands (Mute channels, Sound scaling) and
# TextGrid-area commands work from the same editor: gridId block.
#
# Provenance: Empirical test, 7 May 2026. editor: soundId hung
# indefinitely in TextGridEditor; editor: gridId succeeded for
# both Mute channels and Sound scaling commands.
#
# AMBIGUITY WARNING: If two editors have the same object name,
# Praat cannot distinguish them. Use numeric IDs to avoid this.
#
# EDITOR LIFECYCLE WARNING (empirical, 9 May 2026):
# Extract commands inside an editor block (Extract visible pitch,
# Extract selected sound for overlap, etc.) create new objects.
# After endeditor, the new object may be selected in the Objects
# window. If you removeObject: the extracted object, confirm you
# are NOT removing the original editor object (check IDs). The
# editor may also close after certain extract operations. Use
# defensive re-opens: selectObject: originalId / View & Edit
# before the next editor: block.
#
# ANALYSIS TOGGLE BEHAVIOR (empirical, 9 May 2026):
# Show spectrogram / Show pitch / Show intensity / Show formants /
# Show pulses are TOGGLES. Calling them when analysis is already
# visible turns it OFF. In scripts, prefer the Settings commands
# (which auto-enable the analysis) over Show toggles when you
# need to ensure analysis is active. Alternatively, use
# Show analyses: to set exact on/off state for all five analyses.
#
# Source: Praat manual Scripting 7.1/7.2 + empirical verification

## Editor lifecycle

  Close                                                             [V]
  # Closes the editor window. Use INSIDE an editor:/endeditor block.
  # USAGE PATTERN: When closing an editor at the end of script processing,
  # the editor may already be closed by user action (window dismissed) or
  # by upstream code. Wrap with `nocheck` to tolerate the already-closed
  # state:
  #     editor: gridId
  #         nocheck Close
  #     endeditor
  # This is one of the rare safe uses of `nocheck` — it guards against a
  # specific known-recoverable state, not arbitrary failure. Standard
  # `nocheck` cautions (House Rules, COMMANDS_Universal.txt errata) still
  # apply: do not use `nocheck` for diagnostic branching, and do not chain
  # multiple `nocheck` commands.
  # Provenance: RespTrack annotation workflow, 13 March 2026.

## Sound menu — Preferences

  Sound scaling: scaling_strategy$, height, minimum, maximum        [V-n]
  # Sets how the waveform display is scaled.                        [V-p]
  # scaling_strategy$ MUST be a string (numeric index fails):
  #   "by whole"                                                    [V-n]
  #   "by window"                                                   [V-n]
  #   "by window and channel"                                       [V-n]
  #   "fixed height"                                                [V-n]
  #   "fixed range"                                                 [V-n]
  # height: used when strategy is "fixed height"
  # minimum/maximum: used when strategy is "fixed range"
  # Works in both SoundEditor and TextGridEditor.                   [V-n]
  # Example: Sound scaling: "fixed range", 2.0, -2.0, 2.0
  #
  # NOTE: enum argument MUST be a string — passing a numeric index
  # produces: "Option argument should be a string, not a number."
  # Provenance: Empirical test, 7 May 2026

  Mute channels: channels$                                          [V]
  # channels: NATURALVECTOR (whitespace-separated channel numbers)
  # Mutes the specified channels during playback.
  # Channels are 1-based. Unmuted channels play normally.
  # All channels NOT listed are automatically unmuted.
  # Pass empty string "" to unmute all channels.                    [V-n]
  #
  # Works in both SoundEditor and TextGridEditor.                   [V-n]
  # In TextGridEditor, target with editor: gridId (not soundId).
  #
  # Example: Mute channels 1-3 of a 4-channel Sound+TextGrid:
  #   selectObject: soundId, gridId
  #   View & Edit
  #   editor: gridId
  #       Mute channels: "1 2 3"
  #   endeditor
  #
  # Provenance: SoundArea.cpp line 705; empirical tests 7+9 May 2026

## Pitch menu — method-specific settings (NEW in Praat 6.4.x)

  # Each pitch analysis method has its own settings command with
  # a DIFFERENT parameter set. The filtered methods have 14 params
  # (including "Attenuation at top"); the raw methods have 13 params.
  #
  # These are the REAL pitch settings commands. The generic 5-param
  # "Pitch settings:" is a compatibility shim.

  Pitch settings (filtered autocorrelation): floor, top, unit$, view_from, view_to, drawing_method$, max_candidates, very_accurate$, attenuation_at_top, silence_threshold, voicing_threshold, octave_cost, octave_jump_cost, voiced_unvoiced_cost  [V] [V-p]
  # 14 parameters — for intonation / vocal-fold vibration.
  #   floor:                Pitch floor (Hz)                default: 50
  #   top:                  Pitch top (Hz)                  default: 800
  #   unit:                 "Hertz", "semitones re 1 Hz",   [V-n]
  #                         "mel", "logHertz",
  #                         "semitones re 100 Hz", "ERB"
  #   view_from:            View range low (units)          default: 0 (auto)
  #   view_to:              View range high (units)         default: 0 (auto)
  #   drawing_method:       "Automatic", "Curve", "Speckle" [V-n]
  #   max_candidates:       integer                         default: 15
  #   very_accurate:        "yes" / "no"                    default: "no"
  #   attenuation_at_top:   0–1                             default: 0.03
  #   silence_threshold:    0–1                             default: 0.09
  #   voicing_threshold:    0–1                             default: 0.50
  #   octave_cost:          per octave                      default: 0.055
  #   octave_jump_cost:     0–1                             default: 0.35
  #   voiced_unvoiced_cost: 0–1                             default: 0.14
  #
  # NOTE: "top" is NOT the same as "ceiling". The filtered methods
  # use a Gaussian low-pass filter that attenuates candidates
  # above top/2. Set top much higher than the expected F0 range.
  # See APPENDIX_D §1A for singing voice considerations.
  #
  # Example:
  #   Pitch settings (filtered autocorrelation): 50, 800, "Hertz",
  #   ... 0, 0, "Automatic", 15, "no", 0.03, 0.09, 0.5, 0.055, 0.35, 0.14

  Pitch settings (filtered cross-correlation): [same 14 parameters] [V-p]
  # Identical parameter layout to filtered autocorrelation.
  # Same defaults.

  Pitch settings (raw cross-correlation): floor, ceiling, unit$, view_from, view_to, drawing_method$, max_candidates, very_accurate$, silence_threshold, voicing_threshold, octave_cost, octave_jump_cost, voiced_unvoiced_cost  [V-p]
  # 13 parameters — for voice analysis (jitter, shimmer, HNR).
  #   floor:                Pitch floor (Hz)                default: 75
  #   ceiling:              Pitch ceiling (Hz)              default: 600
  #   unit:                 (same options as filtered)
  #   view_from/to:         View range                      default: 0, 0
  #   drawing_method:       "Automatic", "Curve", "Speckle"
  #   max_candidates:       integer                         default: 15
  #   very_accurate:        "yes" / "no"                    default: "no"
  #   silence_threshold:    0–1                             default: 0.03
  #   voicing_threshold:    0–1                             default: 0.45
  #   octave_cost:          per octave                      default: 0.01
  #   octave_jump_cost:     0–1                             default: 0.35
  #   voiced_unvoiced_cost: 0–1                             default: 0.14
  #
  # NO "Attenuation at top" parameter — raw methods do not filter.
  # NOTE: "ceiling" is a hard cutoff, unlike "top" which is soft.
  # Default path-finding parameters differ from filtered methods.
  #
  # Example:
  #   Pitch settings (raw cross-correlation): 75, 600, "Hertz",
  #   ... 0, 0, "Automatic", 15, "no", 0.03, 0.45, 0.01, 0.35, 0.14

  Pitch settings (raw autocorrelation): [same 13 parameters]        [V-p]
  # Identical parameter layout to raw cross-correlation.
  # Same defaults.

  # COMPATIBILITY COMMAND (generic):
  Pitch settings: floor, ceiling, unit$, method$, drawing_method$   [V]
  # 5 parameters — simplified interface.                            [V-p]
  #   floor:           Pitch floor (Hz)               default: 75
  #   ceiling:         Pitch ceiling/top (Hz)         default: 600
  #   unit:            "Hertz" etc.
  #   method:          "autocorrelation",              [V-n]
  #                    "cross-correlation",
  #                    "filtered autocorrelation",
  #                    "raw autocorrelation",
  #                    "filtered cross-correlation",
  #                    "raw cross-correlation"
  #   drawing_method:  "Automatic", "Curve", "Speckle"
  # This is a compatibility shim. It sets floor, ceiling/top,
  # unit, method, and drawing method, but NOT the advanced
  # path-finding parameters (silence threshold, voicing threshold,
  # etc.). Use the method-specific commands for full control.
  # NOTE: Pitch settings: auto-enables pitch computation.
  # Calling Show pitch is NOT required before Get pitch when
  # Pitch settings: is used.

  How to choose a pitch analysis method                             [S]
  # Opens help page (not useful in scripting).

# This table documents which commands require which analysis to be
# enabled, and how to enable it. Based on empirical testing.
#
#   | Query command       | Requires                  | How to enable              |
#   |---------------------|---------------------------|----------------------------|
#   | Get pitch           | Pitch enabled             | Pitch settings: (any)      |
#   | Get minimum pitch   | Pitch enabled + selection | Pitch settings: + Select:  |
#   | Get maximum pitch   | Pitch enabled + selection | Pitch settings: + Select:  |
#   | Get intensity       | Intensity visible         | Show intensity + settings  |
#   | Get min/max intens  | Intensity visible + sel   | Show intensity + settings  |
#   | Get Nth formant     | Formants visible          | Show formants + settings   |
#   | Get Nth bandwidth   | Formants visible          | Show formants + settings   |
#   | Get jitter (*)      | Pitch + Pulses visible    | Pitch settings + Show pulses |
#   | Get shimmer (*)     | Pitch + Pulses visible    | Pitch settings + Show pulses |
#   | Voice report        | Pitch + Pulses visible    | Pitch settings + Show pulses |
#
# KEY BEHAVIORAL FINDING:
# - Pitch settings: auto-enables pitch (no Show pitch needed)
# - Intensity settings: does NOT auto-enable intensity (Show intensity needed first)
# - Formant settings: does NOT auto-enable formants (Show formants needed first)
# - Show X commands are TOGGLES — calling when already on turns it OFF
#
# RECOMMENDED PATTERN for scripting:
#   editor: soundId
#       Show analyses: "no", "yes", "no", "no", "no"   # exact state
#       Pitch settings (filtered autocorrelation): ...  # set params
#       Move cursor to: 0.5
#       pitch = Get pitch
#   endeditor

# MUTING CHANNELS (the RIP use case):
#
# To open a multi-channel Sound + TextGrid, mute respiratory channels,
# and let the user annotate while hearing only the audio channel:
#
#   selectObject: editorSoundId, gridId
#   View & Edit
#   editor: gridId
#       Mute channels: "1 2 3"
#   endeditor
#   beginPause: "Annotate phases"
#       comment: "Channel 1: Abdomen | Channel 2: Sum | Channel 3: Rib cage"
#       comment: "Channel 4: Audio (unmuted)"
#       comment: ""
#       comment: "Use labels: inh, susp, phon"
#   clicked = endPause: "Stop", "Continue", 2, 0
#   if clicked = 1
#       exitScript: "User stopped."
#   endif
#
# NOTE: editor: gridId (not soundId) — TextGridEditor registers under
# the TextGrid. Using soundId hangs.
# NOTE: beginPause with comment: lines replaces pauseScript for
# multi-line instructions — pauseScript truncates after the first line.
#
# SOUND SCALING (set display range):
#
#   editor: soundId    # or gridId in TextGridEditor
#       Sound scaling: "fixed range", 2.0, -5.0, 5.0
#   endeditor
#
# Scaling strategy must be a string, not a number.
# Valid values: "by whole", "by window", "by window and channel",
#   "fixed height", "fixed range"
#
# QUERY PATTERN:
#
# To query values from the editor (e.g., pitch at cursor):
#
#   editor: soundId
#       Pitch settings (filtered autocorrelation): 50, 800, "Hertz",
#       ... 0, 0, "Automatic", 15, "no", 0.03, 0.09, 0.5, 0.055, 0.35, 0.14
#       Move cursor to: 0.5
#       pitch = Get pitch
#   endeditor
#   appendInfoLine: "Pitch at cursor: ", pitch, " Hz"
