## Photo Object (alpha compositing for Picture window)

  # Photo objects provide pixel-level RGBA control for the Picture window.
  # They are the only way to get true alpha transparency in Praat output.
  # For per-point alpha dots, prefer Insert picture from file: with
  # pre-rendered sprites — Photo has per-object lifecycle overhead.
  # See BEST_PRACTICES_DRAWING.txt for the sprite approach.

  Create simple Photo: name$, nRows, nCols, redFormula$, greenFormula$, blueFormula$
  # Verified: Create simple Photo: "test", 4, 4, ~0, ~0, ~1
  # NOTE: 6 parameters (not 7). Transparency defaults to 0 (opaque).
  # Set separately via Formula (transparency):.
  # Provenance: Empirical test, 21 Mar 2026

  Create Photo: name$, xmin, xmax, nCols, dx, x1, ymin, ymax, nRows, dy, y1, redFormula$, greenFormula$, blueFormula$
  # Verified: Create Photo: "blue", 5, 8, 4, 0.75, 5.375, 2, 5, 4, 0.75, 2.375, ~0, ~0, ~1
  # NOTE: 14 parameters. Domain coordinates (xmin/xmax/ymin/ymax)
  # determine where Paint image: places the image.
  # Provenance: Empirical test, 21 Mar 2026

  Formula (transparency): ~expression
  # Verified: Formula (transparency): ~0.5
  # 0 = opaque, 1 = invisible. Operates on selected Photo.
  # Provenance: Empirical test, 21 Mar 2026

  Paint image: fromX, toX, fromY, toY
  # Verified: Paint image: 0, 0, 0, 0
  # Alpha-composites Photo against existing Picture window content.
  # 0,0,0,0 = entire Photo at its internal domain coordinates.
  # Provenance: Empirical test, 21 Mar 2026

  # ERRATA: Paint image resets Picture window Axes
  # After Paint image:, the Picture window Axes are set to the Photo's
  # internal coordinate domain. Subsequent drawing commands use the
  # Photo's coordinates until Axes: is explicitly restored.
  # Provenance: Scatter plot crash debug, 21 Mar 2026

  # ERRATA: Photo not viable for per-point rendering
  # Per-object lifecycle overhead (Create → Formula → Paint → Remove)
  # makes Photo-based dots ~2× slower than Insert picture from file:
  # with pre-rendered sprites. Paint image: always paints at the Photo's
  # internal domain coords — no domain modification commands exist.
  # Timing: 1500 dots — Paint circle 0.177s, sprites 0.173s, Photo 0.374s.
  # Architecture decision: sprites for per-point alpha, Photo rejected.
  # Provenance: Timing benchmark + visual comparison, 21 Mar 2026
