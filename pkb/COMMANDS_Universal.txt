## Selection Management

  selectObject: id
  selectObject: id1, id2, ...
  # Selects one or more objects by numeric ID. Clears previous selection.
  # Multi-argument form: selectObject: id1, id2, id3
  # Also accepts string form: selectObject: "Sound mySound"
  plusObject: id
  # Adds an object to the current selection.
  # Also accepts string form: plusObject: "Sound mySound"
  minusObject: id
  # Removes an object from the current selection.
  # Also accepts string form: minusObject: "Sound mySound"
  removeObject: id
  removeObject: id1, id2, ...
  # Permanently removes object(s) from the object list.
  # Multi-argument form: removeObject: id1, id2, id3
  select all
  # Selects all objects in the object list. No colon, no arguments.

  # Selection query functions (use in expressions, not as commands):
  #   numberOfSelected ()            — total selected objects
  #   numberOfSelected ("Sound")     — selected objects of a type
  #   selected ("Sound")             — ID of the first selected Sound
  #   selected ("Sound", n)          — ID of the nth selected Sound
  #   selected$ ("Sound")            — name of the first selected Sound
  #   selected$ ("Sound", n)         — name of the nth selected Sound
  #   selected# ("Sound")            — vector of all selected Sound IDs

  # INTERPRETER BEHAVIOR: Selection is a set, not a multiset.
  # Adding an already-selected object via plusObject: is a no-op.
  # Selection always contains each object exactly once.
  # selectObject: a, b, b produces the selection {a, b}, not {a, b, b}.
  # Combine to stereo, Combine to mono, and other multi-selection
  # commands see each selected object once.
  # Provenance: Empirical test + Praat source, 12 Mar 2026

  # Sampled extrema queries (Matrix, Sound, Ltas, and other Sampled types)
  # These are inherited from the Vector/Sampled hierarchy.
  # No-argument forms return the global extremum across the entire object.
  Get minimum
  # Verified: Get minimum (on Matrix created from Ltas and PowerCepstrum)
  Get maximum
  # Verified: Get maximum (on Matrix created from Ltas and PowerCepstrum)
  # Provenance: A2 batch analyzer v2.3+ development, 25 Feb 2026
  #
# NOTE: These no-argument forms differ from the parameterized forms
  # in COMMANDS_Sound.txt (Get minimum: from time, to time, interpolation$).
  # The parameterized forms allow time-range and interpolation control;
  # the no-argument forms return the raw global extremum.
  # Both forms work on Sound objects. Matrix objects support only the
  # no-argument forms.
  #
  # AVOID UNNECESSARY MATRIX CONVERSION:
  # Many object types (Sound, Pitch, Intensity, Ltas, Spectrum) support
  # their own Get minimum / Get maximum queries — either parameterized
  # or no-argument. Converting to Matrix just to call Get minimum /
  # Get maximum is unnecessary overhead when the source object already
  # supports the query. Check the relevant COMMANDS_*.txt file first.
  # Reserve Matrix conversion for object types that genuinely lack
  # min/max queries (e.g., PowerCepstrum before Get peak: was available).

## Syntax Notes and Errata

  # ERRATA: Praat for-loop syntax constraint
  # Praat's for loop supports ONLY: for var from start to end
  # There is NO "by" / "step" clause. Increment is always 1.
  # To step by N, use a multiplier inside a standard for loop:
  #   for x to 10
  #       value = x * step
  #       ... use value ...
  #   endfor
  # Example — iterate 3, 6, 9, ... 30:
  #   for x to 10
  #       appendInfoLine: x * 3
  #   endfor
  # Provenance: Script error confirmed 22 Feb 2026

  # ERRATA: Query commands cannot be used inline in expressions
  # Praat query commands (Get start time, Get end time, Get maximum, etc.)
  # are COMMANDS, not functions. They return values only when used as
  # standalone assignment statements:
  #
  #   WRONG:  .mid = (Get start time + Get end time) / 2
  #   RIGHT:  .s = Get start time
  #           .e = Get end time
  #           .mid = (.s + .e) / 2
  #
  # The error message is: "Unknown symbol «Get» in formula"
  # This applies to ALL query commands across all object types.
  # Provenance: DFT Demo v3.1, confirmed 28 Feb 2026

  # ERRATA: Indexed variables do not resolve inside Formula context
  # Inside Extract rows where: ~self$["col"] = array$[i], the
  # array$[i] is NOT resolved — it fails silently. Assign to a
  # plain variable first:
  #   temp$ = array$[i]
  #   Extract rows where: ~self$["col"] = temp$
  # Provenance: Ping pooling script, 10 Mar 2026

  # ERRATA: nocheck can corrupt interpreter variable state
  # After nocheck on a failing command, subsequent commands may fail
  # silently even if they should succeed. Variables may remain undefined.
  # variableExists() checks don't reliably distinguish "command failed"
  # from "variable never declared" in this state.
  # Implication: Never use nocheck for diagnostic branching. Use separate
  # scripts or file-output diagnostics instead.
  # Provenance: FormantPath diagnostic series, 21 Mar 2026.
  # Confirmed: diagnostic v2 (all nocheck) showed all queries failing;
  # diagnostic v5 (no nocheck, one command) showed same queries passing.
