## 1. PITCH EXTRACTION

# ALGORITHM SELECTION GUIDE
#
# Which pitch algorithm to use depends on the PURPOSE of the analysis.
# Source: Praat manual, "how to choose a pitch analysis method"
# (https://www.fon.hum.uva.nl/praat/manual/how_to_choose_a_pitch_analysis_method.html)
#
#   PURPOSE                              ALGORITHM                    SECTION
#   -----------------------------------  ---------------------------  -------
#   F0 tracking, intonation, prosody,    Filtered autocorrelation     §1A
#     melodic analysis
#
#   Voice quality input (jitter,         Raw cross-correlation        §1B
#     shimmer, HNR via PointProcess)
#     — uses shorter windows for
#       precise period alignment
#
#   Raw periodicity (non-speech          Raw autocorrelation          §1C
#     signals, missing fundamentals)
#
#   Filtered cross-correlation           Not recommended              —
#     — no known preferred use case
#     (Praat manual: experimental)
#
# If the task requires BOTH F0 tracking AND voice quality measures,
# use TWO SEPARATE pitch objects — one per algorithm. Never reuse a
# filtered autocorrelation Pitch for jitter/shimmer input, or a raw
# cross-correlation Pitch for intonation analysis.
#
# If the purpose is ambiguous, ASK THE USER before selecting.

### 1A. Pitch (filtered autocorrelation) — for F0 contour tracking, intonation

# Use case: F0 tracking, prosody, intonation contours
# NOTE: Parameter is "pitch top" (NOT "pitch ceiling")
# NOTE: Requires 11 parameters — "voiced unvoiced cost" is the 11th

To Pitch (filtered autocorrelation): 0.0, 50, 800, 15, "no", 0.03, 0.09, 0.5, 0.055, 0.35, 0.14

# Parameter breakdown (Praat GUI defaults, verified 25 Feb 2026):
#   time step = 0.0 (auto: 0.75 / pitch floor)
#   pitch floor = 50 Hz
#   pitch top = 800 Hz            ← NOT "pitch ceiling"
#   max candidates = 15
#   very accurate = "no"
#   attenuation at top = 0.03
#   silence threshold = 0.09
#   voicing threshold = 0.5
#   octave cost = 0.055
#   octave jump cost = 0.35
#   voiced unvoiced cost = 0.14
#
# CRITICAL — 2× PITCH TOP RULE (FAC only):
# The filtered autocorrelation algorithm applies an internal low-pass
# filter at pitch top. Attenuation begins at pitch_top / 2. This means:
#   - pitch top = 800 → reliable tracking degrades above ~400 Hz
#   - To track F0 at frequency X, set pitch top ≥ 2 × X
#
# Examples:
#   Speech only (typical adult):     pitch floor = 75, pitch top = 600
#   Higher voices (child, soprano):  pitch floor = 50, pitch top = 800
#   Singing up to C5 (523 Hz):      pitch top ≥ 1050 Hz
#   Singing up to C6 (1047 Hz):     pitch top ≥ 2100 Hz
#   Lower voices (bass):            pitch floor = 50, pitch top = 400
#
# The 2× rule applies ONLY to filtered autocorrelation because of its
# internal LPF. Raw cross-correlation and raw autocorrelation use a hard
# cutoff at pitch ceiling — see §1B and §1C.
#
# THESE PARAMETERS DIFFER SUBSTANTIALLY from raw cross-correlation and
# raw autocorrelation defaults. Do NOT copy parameters between algorithms.
#
# Always solicit the expected pitch range from the user before selecting
# pitch floor and pitch top values.

### 1B. Pitch (raw cross-correlation) — for voice quality input (jitter/shimmer)

# Use case: Input to PointProcess for jitter, shimmer, voice quality measures
# NOTE: Parameter is "pitch ceiling" (NOT "pitch top")
# NOTE: Requires 10 parameters (no "attenuation at top")

To Pitch (raw cross-correlation): 0.0, 75, 600, 15, "no", 0.03, 0.45, 0.01, 0.35, 0.14

# Parameter breakdown (Praat GUI defaults, verified 25 Feb 2026):
#   time step = 0.0 (auto)
#   pitch floor = 75 Hz
#   pitch ceiling = 600 Hz        ← NOT "pitch top"; hard cutoff
#   max candidates = 15
#   very accurate = "no"
#   silence threshold = 0.03
#   voicing threshold = 0.45
#   octave cost = 0.01
#   octave jump cost = 0.35
#   voiced unvoiced cost = 0.14
#
# Rationale: raw cross-correlation avoids the spectral weighting that
# filtered autocorrelation applies, giving more accurate period detection
# for perturbation measures.
#
# Pitch ceiling is a HARD CUTOFF — no LPF attenuation. Set it to exceed
# the highest expected F0. For singing tasks, raise pitch ceiling to
# cover the vocal range (e.g., soprano to C6 → pitch ceiling = 1100 Hz).
#
# Always solicit the expected pitch range from the user.

### 1C. Pitch (raw autocorrelation)

# Use case: Alternative to raw cross-correlation; same parameter space
# NOTE: Parameter is "pitch ceiling" (NOT "pitch top")
# NOTE: Requires 10 parameters (no "attenuation at top")

To Pitch (raw autocorrelation): 0.0, 75, 600, 15, "no", 0.03, 0.45, 0.01, 0.35, 0.14

# Parameter breakdown (Praat GUI defaults, verified 25 Feb 2026):
#   time step = 0.0 (auto)
#   pitch floor = 75 Hz
#   pitch ceiling = 600 Hz        ← NOT "pitch top"; hard cutoff
#   max candidates = 15
#   very accurate = "no"
#   silence threshold = 0.03
#   voicing threshold = 0.45
#   octave cost = 0.01
#   octave jump cost = 0.35
#   voiced unvoiced cost = 0.14
#
# Identical defaults to raw cross-correlation (§1B). The algorithms
# differ internally but share the same parameter space and defaults.
#
# Pitch ceiling is a HARD CUTOFF — same guidance as §1B.
# Always solicit the expected pitch range from the user.

## 4. FORMANT ANALYSIS

### 4A. Formant (burg) — general vowel analysis

To Formant (burg): 0.0, 5, 5500, 0.025, 50

# Parameter breakdown (Praat GUI defaults, verified 25 Feb 2026):
#   time step = 0.0 (auto: 0.25 * window length)
#   max number of formants = 5
#   formant ceiling = 5500 Hz    ← Praat default (labeled "= adult female")
#   window length = 0.025 s
#   pre-emphasis from = 50 Hz
#
# FORMANT CEILING SELECTION (hard):
# Formant ceiling must ALWAYS be user-accessible via a form or
# beginPause field. Never hardcode a single value.
#
# Recommended GUI pattern:
#
#   optionmenu: "Vocal tract size", 1
#       option: "Smaller (5500 Hz)"
#       option: "Larger (5000 Hz)"
#
#   # Derive ceiling from selection:
#   if vocal_tract_size = 1
#       formantCeiling = 5500
#   else
#       formantCeiling = 5000
#   endif
#
# The labels "Smaller" and "Larger" refer to vocal tract dimensions
# and avoid gender assumptions. Smaller tracts (typically higher-voiced
# speakers) produce higher formant frequencies.
#
# If the user's protocol specifies a different ceiling, they can
# override via a raw numeric field instead of the optionmenu.

### 4B. Formant (burg) — voice quality / clinical context

## 7. PLAUSIBILITY RANGES

# After querying any acoustic measure, check that the result falls within
# plausible bounds. If outside these ranges, emit a non-blocking warning
# via appendInfoLine. Do NOT exitScript — the user may have valid reasons.

# Measure               Plausible range         Warning if outside
# -----------           ---------------         ------------------
# F0 (Hz)               50 – 1000                "F0 value outside typical speech or singing range"
# Jitter (local, %)     0 – 5                   "Jitter unusually high; check voicing"
# Shimmer (local, %)    0 – 15                  "Shimmer unusually high; check voicing"
# HNR (dB)              -20 – 40                "HNR outside expected range"
# Intensity (dB SPL)    20 – 120                "Intensity outside typical recording range"
# CPPS (dB)             0 – 25                  "CPPS outside expected range"
# Formant F1 (Hz)       150 – 1200              "F1 outside typical range"
# Formant F2 (Hz)       500 – 3500              "F2 outside typical range"
# Formant F3 (Hz)       1500 – 4500             "F3 outside typical range"

# Pattern for plausibility check:
#
#   if hnr <> undefined
#       if hnr < -20 or hnr > 40
#           appendInfoLine: "WARNING: HNR = ", fixed$ (hnr, 2),
#           ... " dB — outside expected range (-20 to 40 dB)."
#       endif
#   endif

## 9. AVQI PARAMETER ISOLATION WARNING

# The Acoustic Voice Quality Index (AVQI) uses a SEPARATE set of analysis
# parameters calibrated together with its regression equation. These
# parameters DIFFER from the clinical defaults in this appendix.
#
# Key differences:
#
#   Parameter          AVQI (Phonanium v02.06)    Clinical (this appendix)
#   ---------------    -----------------------    -----------------------
#   HNR method         Voice report               To Harmonicity (cc)
#   Shimmer units      × 100 (percentage)         Raw proportion (0–1) or offer as percentage (x 100)
#   CPPS subtract tilt "no"                        "yes"
#   CPPS fit method    "Robust"                   "Robust slow"
#   High-pass filter   34 Hz stop Hann band       Not applied
#
# When a script computes BOTH AVQI and per-token clinical measures,
# it must maintain two separate analysis pipelines with their respective
# parameter sets. Never substitute Appendix D values into AVQI computation
# or vice versa.
#
# For the complete AVQI parameter specification, see the Script A2
# handoff document (HANDOFF_A2_Batch_Analyzer.md §4).
