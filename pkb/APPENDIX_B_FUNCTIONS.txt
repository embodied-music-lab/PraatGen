# FORMULA-CONTEXT OBJECT ATTRIBUTES (not functions, but used in Formula: expressions)
# Inside Formula: ~expression, bare attribute names refer to the object
# being modified. These are NOT script-level variables ‚Äî they exist only
# inside Formula expressions.
#
# Attributes:
#   xmin, xmax  ‚Äî time/frequency domain bounds
#   nx, dx      ‚Äî number of samples/frames/bins, step size
#   ymin, ymax  ‚Äî frequency domain bounds (Spectrogram, Cochleagram, Matrix)
#   ny, dy      ‚Äî number of frequency bands, band spacing
#   ncol, nrow  ‚Äî columns/rows (Matrix, TableOfReal, Table)
#   col$[i]     ‚Äî column name (TableOfReal, Table)
#   row$[i]     ‚Äî row name (TableOfReal)
#   self        ‚Äî current cell value
#   Self.xmin   ‚Äî disambiguates from script variable of same name
#                  (capital S, with dot)
#
# Cross-object reference syntax:
#   object[id].xmin          ‚Äî attribute of object by numeric ID
#   object["Sound name"].nx  ‚Äî attribute of object by type + name
#
# Note: self (lowercase, no dot) = current cell value.
#       Self.xmin (capital S, dot) = attribute of object being modified.
#       These are two distinct keywords.
#
# Provenance: Praat manual, Formulas 8: Attributes of objects.
# See also Master Prompt Rule 9 (Formula context exception).
#
