# Parameter notation: $ = string, ? = boolean, others = numeric

# Cross-type dependencies:
#   - Extract Electroglottogram is a Sound command (creates Electroglottogram from Sound)
#   - Derivative and First central difference produce new Electroglottogram objects
#   - To TextGrid (closed glottis) produces a TextGrid
#   - To AmplitudeTier (levels) produces an AmplitudeTier
#
# Availability: Introduced in Praat 6.2.x (djmw). This is a relatively
# recent object type. Scripts targeting older Praat versions should use
# Sound objects with manual channel extraction instead.
#
# IMPORTANT: The Electroglottogram is a distinct object type, NOT a Sound.
# Sound-specific commands (To Pitch, To Formant, etc.) are NOT available
# on Electroglottogram objects. If you need those analyses on an EGG signal,
# keep or create a Sound copy of the channel before extracting as
# Electroglottogram, or extract the channel as Sound using Extract one channel:.

# ============================================================================
# OBJECT DESCRIPTION
# ============================================================================
#
# The Electroglottogram represents changes in vocal fold contact area
# (VFCA) during vocal fold vibration. It is a sampled time-domain signal
# with the same internal structure as Sound (samples, sampling rate, time
# domain), but typed distinctly so that EGG-specific analysis commands
# are available and audio-specific commands are not.
#
# Signal orientation convention:
#   - Positive direction = increasing vocal fold contact area (closing)
#   - Negative direction = decreasing contact area (opening)
#   - Steepest upward slope = glottal closure (fast event)
#   - Gentler downward slope = glottal opening (slower event)
#
# If the raw EGG channel has inverted polarity, use the invert? parameter
# during extraction to correct it.
#
# Waveform landmarks (per Praat manual, after Herbst 2019):
#   a — initial contact of lower vocal fold margins
#   b — upper margins make initial (not full) contact
#   c — maximum vocal fold contact reached
#   d — de-contacting initiated by separation of lower margins
#   e — upper margins start to separate
#   f — glottis open, minimal contact area
#
# Caveat from Herbst (2019): The vocal folds do not vibrate as a uniform
# mass. Phase differences along inferior-superior and anterior-posterior
# dimensions cause time-delayed contacting and de-contacting. There is
# thus no specific instant of glottal closing/opening, but rather an
# interval during which closing/opening occurs.

# ============================================================================
# COMMANDS
# ============================================================================

---

  # Create (from Sound — this is a Sound command, listed here for workflow clarity)
  # Menu path: Sound > Convert > Extract Electroglottogram...
  Extract Electroglottogram: channel number, invert?
  # Source: https://www.fon.hum.uva.nl/praat/manual/Sound__Extract_Electroglottogram___.html
  #
  # channel number: which channel contains the EGG signal (typically 2 in
  #   a stereo recording where ch1 = audio, ch2 = EGG)
  # invert?: "yes" or "no" — invert if the steepest slope is downward
  #   (convention requires steepest slope to be upward = closing)
  #
  # Returns: a new Electroglottogram object (NOT a Sound)
  #
  # NOTE: This command is in the Sound menu, not the Electroglottogram menu.
  # The input must be a selected Sound object (typically multi-channel).

  # Modify (in-place)
  High-pass filter: from frequency, smoothing
  # Source: https://www.fon.hum.uva.nl/praat/manual/Electroglottogram__High-pass_filter___.html
  #
  # Filters away baseline and amplitude drift artifacts.
  # from frequency (Hz): frequencies below this are suppressed
  # smoothing (Hz): width of transition band between suppressed and passed
  #   Frequencies < fromFrequency: fully suppressed
  #   Frequencies > fromFrequency + smoothing: fully passed

  # Convert / Analyze — derivative (DEGG)
  Derivative: low-pass frequency, smoothing, new absolute peak
  # Source: https://www.fon.hum.uva.nl/praat/manual/Electroglottogram__Derivative___.html
  # Algorithm: same as Sound: To Sound (derivative)... — spectral-domain
  #   exact derivative via Fourier multiplication by 2*pi*i*f
  #
  # low-pass frequency (Hz): highest frequency to retain in derivative
  #   (derivative amplifies high frequencies — low-pass filtering essential)
  # smoothing (Hz): transition band width for the low-pass filter
  # new absolute peak: rescale result to this peak amplitude
  #   Use 0.0 to prevent any scaling
  #   Use 0.99 if you want to listen to the DEGG without distortion
  #
  # Returns: a new Electroglottogram object (the DEGG signal)
  #
  # NOTE: This is the mathematically exact derivative. Henrich et al. (2004)
  # and Herbst et al. (2014) use the first central difference approximation
  # instead (see below). Results differ at high frequencies.

  First central difference: new absolute peak
  # Source: https://www.fon.hum.uva.nl/praat/manual/Electroglottogram__First_central_difference___.html
  # Algorithm: (dx/dt)[i] = (x[i+1] - x[i-1]) / (2 * dt)
  #
  # new absolute peak: rescale to this peak amplitude; 0.0 = no scaling
  #
  # Returns: a new Electroglottogram object
  #
  # This is the approximation used by Henrich et al. (2004) and widely
  # adopted in EGG literature. Preferred over Derivative when replicating
  # published DEGG-based methods (contact quotient via DEGG peaks, etc.).

  # Convert / Analyze — glottal timing
  To TextGrid (closed glottis): pitch floor, pitch ceiling, closing threshold, peak threshold
  # Source: https://www.fon.hum.uva.nl/praat/manual/Electroglottogram__To_TextGrid__closed_glottis____.html
  #
  # pitch floor (Hz): cycles with lower pitch are not considered
  # pitch ceiling (Hz): cycles with higher pitch are not considered
  # closing threshold: value between 0 and 1; the closing/opening moments
  #   are found at: valley + closingThreshold * (peak - valley)
  #   Typical value: 0.3 (per Praat manual illustration)
  #   Lower values = closer to the valley (earlier closing, later opening)
  #   Higher values = closer to the peak (later closing, earlier opening)
  # peak threshold: relative amplitude threshold (0–1); cycles whose peaks
  #   are below this fraction of the maximum peak are excluded
  #
  # Returns: a TextGrid with intervals marking closed glottis periods
  #
  # Algorithm: Finds peaks and valleys guided by pitch floor/ceiling,
  # then determines glottal closing and opening times via level crossings
  # at the threshold amplitude.
  #
  # WARNING: See Herbst (2019) caveat above — GCI/GOI are inherently
  # approximate due to non-uniform vocal fold vibration.

  # Convert / Analyze — amplitude levels
  To AmplitudeTier (levels): pitch floor, pitch ceiling, closing threshold, boolean1?, boolean2?
  # Verified: To AmplitudeTier (levels): 75, 500, 0.3, "yes", "yes"
  # Provenance: Paste Commands, user session 22 Feb 2026
  #
  # pitch floor (Hz): cycles with lower pitch are not considered
  # pitch ceiling (Hz): cycles with higher pitch are not considered
  # closing threshold: value between 0 and 1; level crossing criterion
  #   (same as in To TextGrid (closed glottis))
  # boolean1?, boolean2?: exact parameter names unknown — verified values
  #   are "yes", "yes". Likely candidates based on EGG analysis patterns:
  #   include closing levels? / include opening levels? — but this is
  #   speculative. Check Praat GUI labels for authoritative names.
  #
  # Returns: an AmplitudeTier object with amplitude level information
  #   at the determined glottal events.

---

*End of Electroglottogram commands.*


# ============================================================================
# WORKFLOW NOTES AND BEST PRACTICES
# ============================================================================
#
# 1. TYPICAL EGG RECORDING SETUP
#
#    Standard EGG recordings produce a stereo (2-channel) file:
#      Channel 1: acoustic microphone signal
#      Channel 2: EGG signal
#    (Channel assignment varies by equipment — always verify)
#
#    Loading workflow:
#      stereoId = Read from file: filePath$
#      eggId = Extract Electroglottogram: 2, "no"
#      # Keep the Sound for acoustic analysis:
#      selectObject: stereoId
#      audioId = Extract one channel: 1
#      removeObject: stereoId
#
#    If you need Sound-type commands on the EGG (To Pitch, To Formant, etc.),
#    extract the channel as a Sound instead:
#      selectObject: stereoId
#      eggSoundId = Extract one channel: 2
#
# 2. POLARITY CHECK
#
#    The EGG convention is: positive = increasing contact (closing).
#    Closing is faster than opening, so the steepest slope should be upward.
#    If your signal has inverted polarity, set invert? to "yes" during
#    extraction, or the derivative peaks will be inverted.
#
#    Visual check: In the waveform, the sharp transitions should go UP
#    (toward positive). If they go down, the signal is inverted.
#
# 3. HIGH-PASS FILTERING
#
#    EGG signals commonly exhibit baseline drift and low-frequency amplitude
#    modulation artifacts from electrode movement, swallowing, etc.
#    High-pass filtering at ~20–40 Hz removes these while preserving the
#    fundamental frequency content (typically > 50 Hz for adults).
#
#    Recommended: High-pass filter: 20, 10
#    (suppresses below 20 Hz, transition band 10 Hz)
#
# 4. DEGG (DERIVATIVE) ANALYSIS
#
#    The DEGG signal is central to EGG analysis because:
#      - Positive DEGG peaks correspond to glottal closure instants (GCI)
#      - Negative DEGG peaks correspond to glottal opening instants (GOI)
#      - The ratio of open/closed phase durations gives the Open Quotient (OQ)
#        or Contact Quotient (CQ = 1 - OQ)
#
#    Two derivative methods are available:
#      a) Derivative: — exact spectral derivative (mathematically precise)
#      b) First central difference: — discrete approximation (literature standard)
#
#    For replication of published methods (Henrich et al. 2004, Herbst et al.
#    2014, Howard 1995), use First central difference. For general-purpose
#    analysis where mathematical precision matters more than method matching,
#    use Derivative with appropriate low-pass filtering.
#
#    When using Derivative:, the low-pass frequency is critical. The derivative
#    amplifies high-frequency noise dramatically (a 10 kHz component is
#    amplified 100x more than a 100 Hz component). Typical recommendation:
#      Derivative: 5000, 100, 0
#    (low-pass at 5000 Hz, 100 Hz smoothing, no amplitude scaling)
#
# 5. CONTACT QUOTIENT / OPEN QUOTIENT FROM TEXTGRID
#
#    After To TextGrid (closed glottis):, the closed intervals can be used
#    to compute CQ per cycle:
#      CQ = closed_duration / period_duration
#      OQ = 1 - CQ = open_duration / period_duration
#
#    The closing threshold parameter directly affects CQ values:
#      - Threshold 0.25 gives values comparable to threshold-based CQ
#        at the 25% level (CQ25, per Howard 1995)
#      - Threshold 0.3 is the Praat manual default illustration
#      - Threshold 0.5 corresponds to the 50% crossing criterion
#
#    IMPORTANT: CQ values are NOT directly comparable across studies
#    unless the threshold criterion is matched. Always report the
#    threshold used.
#
# 6. PITCH FLOOR/CEILING FOR EGG COMMANDS
#
#    The To TextGrid (closed glottis) and To AmplitudeTier (levels)
#    commands use pitch floor/ceiling to guide cycle detection.
#    Use the same pitch range you would for acoustic pitch analysis:
#      - Adult male speech: 75–300 Hz
#      - Adult female speech: 100–500 Hz
#      - Singing (depends on range): may need 50–1000 Hz or wider
#    Match these to the APPENDIX_D clinical defaults when applicable.
#
# 7. RELATIONSHIP TO SOUND-BASED EGG ANALYSIS
#
#    Before the Electroglottogram type existed, all EGG analysis in Praat
#    was done by treating the EGG channel as a Sound. This approach still
#    works and provides access to the full Sound command set (To Pitch,
#    To PointProcess, etc.). The dedicated Electroglottogram type adds
#    EGG-specific commands but restricts access to Sound commands.
#
#    Choose based on your needs:
#      - Electroglottogram type: for closed-glottis detection, DEGG analysis,
#        and workflows that benefit from type safety
#      - Sound type (via Extract one channel): for pitch tracking from EGG,
#        PointProcess-based period detection, jitter/shimmer from EGG,
#        or when you need commands not available on Electroglottogram
#
#    For comprehensive analysis, extract BOTH:
#      selectObject: stereoId
#      eggObjectId = Extract Electroglottogram: 2, "no"
#      selectObject: stereoId
#      eggSoundId = Extract one channel: 2
#
# 8. EXTERNAL TOOLS AND SCRIPTS
#
#    Several community tools extend Praat's EGG capabilities:
#      - praatdet (James Kirby): OQ measurement via DEGG peak detection
#        with manual correction interface. Uses PointProcess for period
#        detection. https://github.com/kirbyj/praatdet
#      - PrEgg (Chan et al. 2024): Extracts CQ, Skew Quotient (SQ),
#        and Peak Increase in Contact (PIC). JASA abstract.
#        https://github.com/maypychan/praat-egg
#      - EGG_DEGG scripts (Marc Brunelle / NC State): Calculate DEGG
#        from EGG channel, combine into stereo for visualization.
#
