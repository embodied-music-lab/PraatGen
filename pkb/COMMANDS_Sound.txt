  #
  # CRITICAL NOTE on pitch floor parameter:
  # Pitch floor controls the internal intensity smoothing window:
  #   window = 6.4 / pitch_floor seconds
  #
  #   pitch_floor = 50 Hz  → 128 ms window (may fill short gaps)
  #   pitch_floor = 75 Hz  →  85 ms window
  #   pitch_floor = 100 Hz →  64 ms window (Praat dialog default)
  #
  # If silence detection fails to split events with a visible gap,
  # check the pitch floor FIRST — increasing it shortens the smoothing
  # window and may resolve the issue without changing threshold or
  # duration parameters.
  #
  # Provenance: Empirical root-cause analysis, A2 v2.6→v2.7, 24 Feb 2026.


  To Pitch (filtered autocorrelation): time step, pitch floor, pitch top, maximum number of candidates, very accurate?, attenuation at top, silence threshold, voicing threshold, octave cost, octave jump cost, voiced unvoiced cost

  # NOTE: "pitch top" controls an internal low-pass filter that attenuates
  # energy from pitch_top/2 upward BEFORE autocorrelation. For reliable
  # tracking, pitch top must be >= 2× the highest F0 you need to track.
  # Speech default: 500–600 Hz. Singing: 2× highest expected F0
  # (e.g., soprano to C6 → pitch top >= 2100 Hz).
  # See APPENDIX_D §1A for canonical parameter sets.
